# __Markdown_Report__               *2026.03.16 C팀 양희재*

## Markdown 보고서 등장배경

요즘은 다양한 생성형 AI를 활용하게 되면서 Markdown 언어로 표현된 문서를 많이 접할 수 있게 되었고 이러한 markdown은 어떻게 작성되기에 일반 텍스트로 표현하는 것과는 다른지 알아보겠습니다. 
Markdown은 간단한 문법을 사용하여 가독성이 높은 문서를 작성할 수 있으며, 작성된 문서를 HTML과 같은 웹 문서 형식으로 쉽게 변환할 수 있다는 점에서 웹 개발 및 기술 문서 작성 분야에서 널리 활용되고 있습니다.
특히 개발 분야에서 프로젝트 문서화, 협업 문서 작성, 기술 블로그 등 다양한 환경에서 Markdown이 사용되고 있기때문에 대한 기본적인 이해는 매우 중요합니다.

따라서 본 보고서는 Markdown의 개념과 특징, 주요 문법 및 활용 사례를 정리하여 Markdown의 활용성과 실용성을 이해하고 전반적인 이해를 돕는것을 목적으로 하며 읽으면서 익힐수 있도록 Markdown문법으로 작성하였습니다.

## Ⅰ. Markdown이란

**Markdown**은 `perl(practical extraction and report language)`기반으로 작성된 일반 텍스트를 **HTML** 형식으로 변환할 수 있도록 만들어진 경량 마크업 언어로 웹 작성자들이 복잡한 HTML 태그를 사용하지 않고도 구조적으로 문서를 작성할 수 있도록 도와주는 변환도구입니다.

여기서 **마크업**이란 문서에 붙이는 설명표입니다.
도서관에서 책을 찾을때 분류를 알려주는 표시를 보듯이 웹 브라우저에서 해당 텍스트가 제목, 문단, 목록 등 어떠한 역할을 하는지 알려주는 표시입니다.


Markdown은 두 가지 의미를 가집니다.

1. 문서를 작성하기 위한 **텍스트 구문 규칙(Formatting Syntax)**
2. perl 기반 일반 텍스트로 쓰여진 문서를 **HTML 형식으로 변환하는 소프트웨어**

Markdown은 간단한 문법과 높은 가독성 덕분에 응용 소프트웨어와 함께 배포되는 **README** 파일이나 온라인 게시물, 협업 문서 등 다양한 환경에서 널리 사용되고 있습니다.

---

## Ⅱ. Markdown의 설계 목적
Markdown의 가장 중요한 설계 목표는 **가독성(Readability)** 입니다.
Markdown 문서는 HTML로 변환하지 않더라도 일반 텍스트만으로도 충분히 읽기 쉽도록 설계되었습니다.
또한 활용가능성의 측면에서 작성자가 복잡한 HTML 태그 없이도 구조적인 문서를 만들 수 있도록 단순한 문법을 제공합니다.

---

## Ⅲ. Markdown의 주요 특징

| 특징 | 설명 |
|---|---|
| 가독성 | 일반 텍스트 형태로도 읽기 쉬움 |
| 편리성 | HTML 태그 없이 문서 작성 가능 |
| 호환성 | 다양한 플랫폼과 에디터에서 지원 가능|
| 변환성 | Markdown 문서를 HTML로 변환 가능 |
| 활용성 | 블로그, 개발 문서, 협업 도구 등에서 활용 가능 |
| 오픈소스 | BSD 라이선스로 무료 사용 가능 |

---

## Ⅳ. Markdown 사용 환경

Markdown은 다양한 플랫폼에서 사용될 수 있습니다.


### ⅰ) Movable Type
Movable Type 2.6 이상에서 Markdown을 사용할 수 있습니다.  
플러그인 형태로 설치하며 게시글 작성 시 Markdown을 선택하면 게시 시 HTML로 자동 변환됩니다.
또한 SmartyPants(python)와 함께 사용하면 다음과 같은 기능을 자동으로 처리할 수 있습니다.

- 스마트 따옴표
- em-dash
- 줄임표(ellipsis)

---

### ⅱ) Blosxom
Blosxom 2.0 이상에서도 Markdown을 사용할 수 있습니다.  
플러그인 파일을 설치하면 블로그 게시글이 자동으로 Markdown 방식으로 처리됩니다.
Meta 플러그인을 함께 사용하면 특정 게시글에만 Markdown을 적용할 수도 있습니다.

---

### ⅲ) BBEdit
Markdown은 Mac OS 환경의 BBEdit 편집기에서도 사용할 수 있습니다.  
BBEdit의 Filters 기능을 이용하여 Markdown 문서를 HTML 형식으로 변환할 수 있습니다.

---

## Ⅴ. Markdown 문법

### ⅰ)  제목 (Headers)
\
제목은 # 기호를 사용하여 만들 수 있습니다. #의 개수가 늘어나면 제목의 크기는 작아집니다. 폰트의 크기 조절용도로 사용하기도 합니다. 아래는 예시입니다.

**※주의: # 기호 다음에는 반드시 공백을 넣어야합니다.**
        
# 제목 1 (H1) 
## 제목 2 (H2)
### 제목 3 (H3)
#### 제목 4 (H4)
##### 제목 5 (H5)
---
        
### ⅱ)  강조 (Emphasis)
\
텍스트를 굵게, 기울임꼴, 취소선으로 표현할 수 있습니다.
 *굵은 글씨** 또는 __굵은 글씨__
 \
*기울임 글씨* 또는 _기울임 글씨_
\
~~취소선~~
\
***굵은 기울임***
\
**굵은 글씨 안에 *기울임 글씨* 포함**
---
### ⅲ) 목록 (Lists)
\
순서가 있는 목록과 순서가 없는 목록을 만들 수 있습니다.
        
#### 순서 없는 목록 (Unordered List)

        - 첫 번째 항목
        - 두 번째 항목
            - 중첩된 항목
            - 또 다른 중첩 항목
        - 세 번째 항목

        * 별표로도 가능
        + 플러스로도 가능
        
---        
### 순서 있는 목록 (Ordered List)
    
    1. 첫 번째 항목
    2. 두 번째 항목
        1. 중첩된 순서 목록
        2. 또 다른 중첩 항목
    3. 세 번째 항목

    1. 숫자가 틀려도
    3. 자동으로
    7. 순서대로 표시됩니다
---    
### ⅳ) 링크 (Links)
\
[링크 텍스트](https://www.example.com)

[제목이 있는 링크](https://www.example.com "링크 제목")
  
[참조 링크][reference]
    
[reference]: https://www.example.com "참조 링크 제목"

---

### ⅴ) 이미지 (Images)
\
![대체 텍스트](https://via.placeholder.com/300x200)

![제목이 있는 이미지](https://via.placeholder.com/300x200 "이미지 제목")

[![링크가 있는 이미지](https://via.placeholder.com/150x100)](https://www.example.com)
---
### ⅵ) 코드 (Code)
\
인라인 코드와 코드 블록을 표시할 수 있습니다.
문장 안에 `인라인 코드`를 넣을 수 있습니다.

`console.log("Hello World!");`
---
### ⅶ) 인용문 (Blockquotes)
\
`>` 기호를 사용하여 인용문을 만들 수 있습니다.

> 이것은 인용문입니다.
> 여러 줄로 작성할 수 있습니다.

> 중첩된 인용문도 가능합니다.
>> 더 깊은 인용문
>>> 최대한 깊은 인용문

> **인용문 안에서도**
> *다른 마크다운 문법*을
> 사용할 수 있습니다.
---
### ⅷ) 테이블 (Tables)
\
| 기호를 사용하여 테이블을 만들 수 있습니다.
| 이름   | 나이 | 직업     |
|--------|------|----------|
| 홍길동 | 25   | 개발자   |
| 김철수 | 30   | 디자이너 |
| 박영희 | 28   | 기획자   |

| 왼쪽 정렬 | 가운데 정렬 | 오른쪽 정렬 |
|:----------|:-----------:|------------:|
| 텍스트    |   텍스트    |      텍스트 |
| 내용      |    내용     |        내용 |
---
### ⅸ) 수평선 (Horizontal Rules)
3개 이상의 하이픈, 별표, 언더스코어로 수평선을 만들 수 있습니다.

## Ⅵ. Markdown 작성요령
1. 문단 사이에는 빈 줄을 넣습니다.
2. 목록 항목은 일관된 기호를 사용합니다.
3. 코드 블록에는 언어를 명시합니다.
3. 링크에는 의미 있는 제목을 붙입니다.
2. 이미지에는 대체 텍스트를 작성합니다.
1. 테이블은 읽기 쉽게 정렬합니다.
